Chapter 04 — All Levels

Git Commands Reference

The commands you'll use every day, organized by what you're trying to do.

Getting Started

Commands for setting up Git and creating repositories.

Command Description Example
git init Create a new repository git init my-project
git clone <url> Copy an existing repository git clone https://github.com/user/repo
git config --global user.name Set your name git config --global user.name "Jane"
git config --global user.email Set your email git config --global user.email "jane@example.com"
git config --list Show all configuration —
git help <command> Show help for a command git help commit

Making Changes

Commands for staging, committing, and inspecting changes.

Command Description Example
git status See what changed —
git add <file> Stage a specific file git add index.html
git add . Stage all changes —
git add -p Interactive staging (choose hunks) Stage parts of a file selectively
git commit -m "msg" Commit staged changes git commit -m "feat: add login"
git commit --amend Modify the last commit Add forgotten files to last commit
git diff See unstaged changes —
git diff --staged See staged changes —
git diff main..feature Compare branches See what's different

Branching

Commands for creating, switching, and managing branches.

Command Description Example
git branch List local branches —
git branch -a List all branches (local + remote) —
git branch <name> Create a new branch git branch feature/auth
git checkout -b <name> Create and switch to a branch git checkout -b feature/auth
git switch <branch> Switch to a branch git switch main
git merge <branch> Merge a branch into current git merge feature/auth
git branch -d <name> Delete a merged branch git branch -d feature/auth
git branch -D <name> Force delete a branch git branch -D feature/old

Syncing

Commands for working with remote repositories.

Command Description Example
git remote add origin <url> Add a remote git remote add origin github.com/user/repo
git push origin <branch> Upload changes git push origin main
git push -u origin <branch> Push and set upstream Future pushes need no arguments
git pull Download and merge updates Same as git fetch && git merge
git pull --rebase Pull with rebase instead of merge Cleaner history
git fetch Check for updates (no merge) See what's new without changing your code
git push origin --tags Push all tags —

Undoing Things

Commands for reverting changes and recovering from mistakes.

Command Description Example
git restore <file> Undo file changes (working dir) git restore index.html
git restore --staged <file> Unstage a file git restore --staged index.html
git reset HEAD~1 Undo last commit (keep changes) Changes go back to working dir
git reset --soft HEAD~1 Undo commit (keep staged) Changes stay staged
git reset --hard HEAD~1 Undo commit (discard changes) Warning: changes are lost
git stash Temporarily shelve changes Switch branches without committing
git stash pop Restore stashed changes Apply and remove from stash
git stash list List all stashes —

Inspection

Commands for viewing history and understanding changes.

Command Description Example
git log View commit history —
git log --oneline Compact history view One line per commit
git log --graph Visual branch graph See merge structure
git log --since="2 weeks ago" Filter by date Recent commits only
git blame <file> Who changed what line Find who wrote a specific line
git tag -a v1.0 Create a release tag git tag -a v1.0 -m "First release"
git show <commit> Show commit details git show a3f7b2c
